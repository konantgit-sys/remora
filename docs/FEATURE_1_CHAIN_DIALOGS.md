# Remora #1 — Chain-диалоги

**Статус:** ✅ Реализовано и протестировано

**Дата завершения:** 2026-05-10 17:45 MSK

**Задача:** Когда кто-то отвечает на ответ Remora, демон должен автоматически продолжить диалог (цепочка).

---

## Архитектура

### Новые методы в `RemoraAdapter`

```python
fetch_replies_to_post(post_id: str, since_sec_ago: int = 86400, limit: int = 30) -> list
```
- Сканирует релеи на поиск реплаев на конкретный пост
- Фильтрует по e-тегу (Nostr протокол для реплаев)
- Возвращает список реплаев с полной информацией

### Новые методы в `Memory`

```python
get_candidate_chain_conversations(max_age_hours: int = 48) -> list
```
- Находит активные диалоги без ответов от автора
- Смотрит в таблицу `remora_reactions` — если нет реплаев, диалог готов к цепочке
- Возвращает top 50 кандидатов

```python
mark_chain_reply(parent_conv_id: int, chain_reply_id: str)
```
- Отмечает в БД, что на диалог был ответ
- Обновляет `remora_reply_id` (для отслеживания цепочек)

```python
get_conversation_by_reply_id(reply_id: str) -> dict
```
- Находит диалог по ID нашего поста (для контекста цепочки)

### Новый метод в `Generator`

```python
generate(candidate: dict, mode: str = None, is_chain: bool = False) -> dict
```
- Параметр `is_chain=True` активирует режим цепочки
- Использует существующий метод `continue_conversation()` для более естественного продолжения
- Повторяет **тон** из оригинального ответа (не выбирает новый)

### Новый цикл в `RemoraDaemon`

Добавлен метод `_chain_phase()` в основной цикл:

```python
# После публикации основных ответов
chain_published = self._chain_phase()

# Логирует количество цепочек за цикл
logger.info(f"🔗 Chain phase: {chain_published} replies published")
```

**Алгоритм:**
1. Находит top-3 диалога без ответов (чтобы не спамить)
2. Сканирует релеи на реплаи к каждому диалогу
3. Берёт первый реплай от оригинального автора
4. Генерирует ответ на цепочку (с тем же тоном)
5. Публикует на релеи

---

## Ограничения Rate-Limiter

- Цепочка использует ту же queue rate-limiter, что и основные ответы
- Max 3 цепочки за цикл (по дизайну, чтобы не перегружать)
- Интервал между цепочкой и основным ответом: `rate_limit_minutes` (по умолчанию 3 мин)

---

## Логирование

Новые логи в `remora.log`:

```
🔗 Chain candidate: abc123... replied to xyz789...
✅ Chain reply: [medium/analytical]
🔗 Chain phase: 2 replies published
```

---

## Что это даёт

- **Engagement**: Если человек ответит на пост Remora, демон ответит обратно → диалог
- **Natural flow**: Используется тот же тон, что и в оригинале → coherent conversations
- **No spam**: Max 3 цепочки за цикл → не раздражает авторов
- **Automatic**: Полностью autonomic, без вмешательства

3. Демон сгенерирует и опубликует ответ

**Метрика успеха:** Когда первый реплай на реплай будет опубликован в Nostr.
