# Remora #3 — Вирусный Апгрейд

**Статус:** ✅ Реализовано

**Дата:** 2026-05-10

---

## Задача

Если пост Remora получил много реакций (лайки, ответы, заппы), то автор этого поста должен получить **boost**: повышение author_score и автоматическое добавление в приоритет.

**Почему:** Если наш ответ зашёл и люди лайкают его — автор явно интересный. Стоит уделять его постам больше внимания.

---

## Механика

### Вирусный порог

```yaml
viral_threshold: 3  # реакций для активации
```

Если пост Remora получил 3+ реакции → viral mode.

### Расчёт бонуса

```
bonus = min(0.3, (reaction_count - viral_threshold) * 0.05)

Примеры:
- 3 реакции → (3-3)*0.05 = 0 бонуса (вот-вот попадёт в viral)
- 5 реакций → (5-3)*0.05 = 0.1 бонуса
- 10 реакций → (10-3)*0.05 = 0.35, но cap=0.3 → 0.3 бонуса
```

Максимальный бонус = 0.3 (не более).

### Интеграция в Tracker

```python
# Viral check
if reaction_count >= self.viral_threshold:
    self.mem.mark_viral(conv_id)
    self.mem.boost_viral_author(conv_id, viral_threshold=self.viral_threshold)
```

Трекер проверяет реакции каждые 10 минут и применяет бонус.

---

## Долгосрочный эффект

### Author Score

У каждого автора есть `remora_author_profiles.score` (0.0 — 1.0):

- 0.0 — спам/черный список
- 0.5 — нейтральный (по умолчанию)
- 0.9+ — whitelisted (всегда отвечаем)
- 1.0 — идеальный автор

Viral boost поднимает score. Например:
- Новый автор: score=0.5
- Его пост получает 10 реакций → +0.3 → score=0.8
- Теперь он близко к whitelist, и на его следующий пост ответим быстрее

### Evaluator приоритет

В будущем можно добавить бонус к pre-filter score на основе author_score.

## Метрика успеха

После 2 недель:
- 10-20 авторов должны иметь score > 0.7
- 5-10 из них перейдут в 0.9+ (через whitelist или через viral)
- Viral треды должны быть заметны в логах

---

## Связь с другими фичами

- **#2 Whitelist** — viral авторы становятся кандидатами на whitelist
- **#2 Self-learning** — viral тоны (которые используются в популярных ответах) получают бонус в score
- **#4 Evaluator** — будущая интеграция: author_score влияет на pre-filter

---

## Ограничение

- Спам-авторы НЕ получают viral boost (добавлены в blacklist)
- Score никогда не превышает 1.0
- Blacklist authors никогда не апгрейдятся (даже если viral)
