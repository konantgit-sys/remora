## Архитектура

###策略: 70-20-10 (Exploitation-Probabilistic-Exploration)

Каждый цикл `get_best_tone()` выбирает тон по стратегии:

1. **70% Exploitation** — берём тон с максимальным score
   - Score = reactions / max(1, uses)
   - Если playful получил 5 реакций из 7 использований → score = 0.71
   - Если analytical получил 2 реакции из 10 → score = 0.20
   - Выигрывает playful

2. **20% Probabilistic** — рулетка по score (weighted random)
   - Вероятность выбрать тон пропорциональна его score
   - Даёт шанс неплохим тонам (score > 0.3) попробоваться чаще

3. **10% Exploration** — самый неиспользованный тон
   - Гарантирует что все 8 тонов хотя бы пробуются
   - Даже "плохие" тоны могут вдруг оказаться лучше в другом контексте

### Затухание (Decay)

`record_tone_use()` применяет decay к score если тон не использовался 24+ часа:

```
decay = max(0.5, 1.0 - (hours_ago / 240))

24 часа  → decay = 0.9  (score *= 0.9)
120 часов → decay = 0.5  (score *= 0.5)
```

**Почему:** Если 3 дня не используем тон, его score падает вполовину. Это позволяет "забыть" плохие периоды и начать с чистого листа.

### Реакции автоматически обновляют score

Когда приходит `kind:7` (реакция/лайк) на пост Remora:
- Tracker находит соответствующий диалог
- `save_reaction()` обновляет tone_stats
- Score пересчитывается

```python
new_reactions = reactions + 1
new_score = clamp(new_reactions / max(1, uses), 0, 1)
```

## БД структура

### Таблица `remora_tone_stats`

| Колонка | Тип | Описание |
|---------|-----|---------|
| tone | TEXT | Имя тона (playful, deadpan, ...) |
| uses | INTEGER | Сколько раз этот тон использовался |
| reactions | INTEGER | Сколько реакций получил |
| score | REAL | reactions / uses (0.0 — 1.0) |
| last_used | TIMESTAMP | Когда последний раз использовался |

### Инициализация

При старте демона — все 8 тонов создаются с `score=0.5, uses=0`:

```python
ALL_TONES = [
    "playful", "deadpan", "absurdist", "wholesome",
    "analytical", "cynical", "mystical", "blunt",
]
```

## Метрики успеха

После недели работы Remora:
1. Один-два тона должны подняться выше 0.6 (хорошие)
2. Один-два упасть ниже 0.3 (плохие)
3. Остальные — флюктуировать около 0.45

**В идеале:** Если нет реакций вообще (engagement=0), то все тоны остаются на ~0.5 с медленным decay.

## Интеграция с Chain-диалогами (#1)

При публикации цепочки — тон берётся из оригинального диалога:

```python
tone = candidate.get("tone_used", "playful")  # Повторяем тон, не выбираем новый
```

Это означает что score ци цепочка тоже влияет на исходный тон (если цепочку лайкнули — score тона растёт ещё больше).
